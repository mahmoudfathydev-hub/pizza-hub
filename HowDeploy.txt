Create a new file called `vercel.json` in your project root:

```json

  "installCommand": "npm install",
  "framework": "nextjs"
```


## üóÑÔ∏è Step 2: Set Up Neon Database

### 2.1 Create Neon Database
1. Go to [neon.tech](https://neon.tech) and log in
2. Click "New Project"
3. Give it a name: `pizzahub-db`
4. Choose a region (pick one close to you)
5. Click "Create Project"

### 2.2 Get Your Database URL
1. After creation, you'll see your dashboard
2. Click "Connection details"
3. Copy the "Connection string" - it looks like:
   ```
   postgresql://username:password@ep-xxx.us-east-2.aws.neon.tech/dbname?sslmode=require
   ```

### 2.3 Update Your Local Database
1. Open your `.env.local` file (create if it doesn't exist)
2. Add your Neon database URL:

```env
DATABASE_URL="your-neon-connection-string-here"
```

### 2.4 Set Up Database Tables
Run these commands in your terminal:

```bash
# Generate Prisma client
npx prisma generate

# Push your database schema to Neon
npx prisma db push
```

If successful, you'll see: "Database schema is up to date"

## üîê Step 3: Set Up Environment Variables

### 3.1 Create Environment Variables File
Create a file called `.env.local` in your project root:

```env
# Database (from Neon)
DATABASE_URL="your-neon-connection-string-here"

# NextAuth - Generate secrets below
NEXTAUTH_URL="http://localhost:3000"
NEXTAUTH_SECRET="your-secret-key-here"

# Cloudinary - Get from cloudinary.com
CLOUDINARY_CLOUD_NAME="your-cloud-name"
CLOUDINARY_API_KEY="your-api-key"
CLOUDINARY_API_SECRET="your-api-secret"
```

### 3.2 Generate NextAuth Secret
Run this command to generate a secure secret:

```bash
openssl rand -base64 32
```

Copy the output and paste it as `NEXTAUTH_SECRET`

### 3.3 Get Cloudinary Keys (Optional)
If you want image uploads:
1. Go to [cloudinary.com](https://cloudinary.com)
2. Sign up for free account
3. Get your Cloud name, API Key, and API Secret from dashboard
4. Add them to your `.env.local`

## üöÄ Step 5: Deploy to Vercel

### 5.1 Import Your Project
1. Go to [vercel.com](https://vercel.com)
2. Click "Add New..." ‚Üí "Project"
3. Find your `pizzahub-food-ordering` repository
4. Click "Import"

### 5.2 Configure Vercel Project
Vercel will auto-detect Next.js. You'll see:

```
Project Name: pizzahub-food-ordering
Framework Preset: Next.js
Root Directory: ./
Build Command: npm run build
Output Directory: .next
Install Command: npm install
```

Click "Continue".

### 5.3 Add Environment Variables
This is important! Add your production environment variables:

1. Click "Environment Variables"
2. Add these one by one:

```
DATABASE_URL = your-neon-connection-string-here
NEXTAUTH_URL = https://pizzahub-food-ordering.vercel.app
NEXTAUTH_SECRET = your-secret-key-here
CLOUDINARY_CLOUD_NAME = your-cloud-name
CLOUDINARY_API_KEY = your-api-key
CLOUDINARY_API_SECRET = your-api-secret
```

**Important**: Use the same secrets from your `.env.local` file, but update `NEXTAUTH_URL` to your Vercel URL.

### 5.4 Deploy!
Click "Deploy" and wait. Vercel will:
1. Install your dependencies
2. Build your Next.js app
3. Deploy it to their global network

This takes 2-5 minutes.

## üõ†Ô∏è Common Problems & Solutions

### Problem 1: "Database connection failed"
**Solution**: 
- Check your `DATABASE_URL` in Vercel environment variables
- Make sure it's the exact same as your Neon connection string
- Redeploy after fixing

### Problem 2: "NextAuth error"
**Solution**:
- Make sure `NEXTAUTH_SECRET` is at least 32 characters
- Check that `NEXTAUTH_URL` matches your Vercel URL exactly
- Redeploy after fixing

### Problem 3: "Build failed"
**Solution**:
- Check the build logs in Vercel dashboard
- Make sure all your environment variables are set
- Try running `npm run build` locally first

### Problem 4: "Images not uploading"
**Solution**:
- Verify Cloudinary credentials are correct
- Check Cloudinary dashboard for upload errors
- Make sure image URLs are in your `next.config.js`
